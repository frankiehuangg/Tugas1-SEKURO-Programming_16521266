## Pendahuluan
Git dan Github adalah 2 benda yang berbeda. Untuk menggunakan Git, kita tidak perlu menggunakan Github; dan begitu pula sebaliknya.

# Version Control System
Git adalah salah satu contoh dari VCS (Version Control System). Version Control System adalah sistem yang mengelola perubahan dari sebuah dokumen, program komputer, website, dan kumpulan informasi lain.

Git lebih digunakan sebagai VCS daripada Google Drive dan Dropbox karena fungsi utama Dropbox dan Google Drive adalah untuk menyimpan data yang bukan *source-code*, ketimbang Git.

Singkatnya, VCS adalah suatu sistem yang menyimpan "rekaman/*snapshot*" perubahan pada source code. VCS memungkinkan kita untuk berkolaborasi dengan lebih baik dan mengetahui siapa yang merubah dan kapan sebuah perubahan terjadi. Terakhir, VCS juga memungkinkan kita untuk kembali ke keadaan sebelum perubahan (*checkout*).

## Apa itu Git?
Git adalah sebuah VCS terdistribusi untuk mengelola perubahan file di dalam folder. Singkatnya, Git adalah sebuah software untuk mengelola perubahan file di dalam folder. Folder dimana file tersebut berada disebut dengan *repository / repo*. Git akan menyimpan semua perubahan yang terjadi dengan perintah *commit*. 

## Mengapa kita menggunakan Git?
- Bisa melacak versi/perubahan pada suatu file.
- Mempermudah untuk melakukan kolaborasi.
- Membagikan kode yang telah dibuat kepada orang lain.

Jika kita tidak menggunakan git, kita perlu membuat sebuah file baru setiap kali kita melakukan perubahan pada suatu file. Dengan adanya git, kita hanya perlu melakukan *commit* setiap kali perubahan file terjadi dan hanya ada 1 file yang tersimpan. Setiap kali kita melakukan *commit*, Git akan membuat sebuah *hash*, yaitu sebuah string penanda agar Git dapat mengetrack semua perubahan. Selain itu, Git juga akan mencatat siapa yang melakukan *commit* tersebut dan kapan *commit* tersebut dilakukan.

## Membuat perubahan pada Git
Saat kita akan mengetes fitur baru, kita akan membuat sebuah cabang (*branch*) agar file utama tidak diubah. Jika branch tersebut fiturnya akan digunakan, kita akan melakukan penggabungan(*merge*) kedua branch tersebut.

## Github
Github adalah sebuah layanan cloud untuk menyimpan & mengelola project / *repo* git dimana semua perubahan akan disimpan dalam cloud (bukan lokal). 
Source code dapat dikirimkan ke Github (*push*) dan didownload dari Github (*pull*). Pada proses *push* dan *pull*, kita akan mengirim *commit* yang berisi perubahan-perubahan pada file yang ada.
Sebelum melakukan *push* dan *pull*, kita harus membuat Github menjadi *remote*, yaitu sumber dari *repo* suatu source code. *Repo* tersebut kemudian akan di *clone* sehingga source code juga tersedia di mesin kita.

## Istilah Git
- repo      = folder yang telah diinisiasi git
- commit    = rekaman/snapshot dari repo kita
- hash      = penanda unik pada sebuah commit berisi string random
- checkout  = berpindah ke sebuah commit
- branch    = cabang dari sebuah commit
- merge     = menggabungkan branch
- remote    = sumber yang memiliki repo
- clone     = mengambil repo dari remote
- push      = mengirimkan commit ke repo
- pull      = mengambil commit dari repo
