## Definisi Branch
Branch adalah sebuah cabang yang independen dari sebuah cabang utama. Fungsi dari Branching antara lain:
- Mengetes fitur/kode yang bersifat eksperimental (belum diketahui apakah bisa digunakan).
- Membuat snapshot tanpa mengganggu cabang utama (master branch).
- Mempermudah pengerjaan kode oleh 2 orang atau lebih secara bersamaan.
Setelah dilakukan branch, umumnya akan dilakukan kembali merge untuk menggabungkan kode kembali ke cabang utama.

## Membuat Branch Baru dengan Commit di Github
Langkah-langkah membuat branch baru saat melakukan sebuah commit:
1. Pada halaman repo, klik **Create new file**.
2. Masukkan isi file.
3. Masukkan pesan commit, kemudian berikan deskripsi untuk penjelasan lebih lanjut.
4. Klik **Create a new branch for this commit and start a pull request.**
5. Klik **Commit changes**

## Membuat Branch Baru pada Halaman Utama Repo
Langkah-langkah membuat branch baru pada halaman utama repo:
1. Pada halaman repo, klik **Branch: master**.
2. Pada input text yang tersedia, buat nama branch baru yang akan dibuat
3. Klik **Create branch: [nama branch] from 'master'**.

## Membuat File Baru Pada Branch Lain
Ketika membuat file baru pada branch lain, file tersebut tidak akan ditemukan pada file lainnya sebelum dilakukan merge. Untuk melihat sejarah branch, dapat dilihat pada menu **Insights**, kemudian **Network**.

## Merging
Langkah-langkah melakukan Merging:
1. Pada halaman repo, pilih branch lain yang akan dimerge menuju branch utama.
2. Klik **Compare & pull request**.
3. Masukkan pesan dan comment pada pull request untuk dibaca oleh pemilik master branch, kemudian klik **Create pull request**.
4. Kembali pada halaman repo, klik **Pull requests**. 
5. Klik **Merge pull request**, kemudian klik **Confirm merge**.
Setelah dilakukan merge, data dari branch lain akan digabungkan kembali ke branch utama.

## Menghapus Branch
Langkah-langkah menghapus branch:
1. Pada halaman repo, klik **branches**.
2. Pilih branch yang akan dihapus, kemudian klik tombol tong sampah **delete**.

## Mengatasi Merge Conflict
Langkah-langkah mengatasi merge conflict:
1. Setelah melakukan pull request, klik **Pull requests** pada halaman repo.
2. Klik **Resolve conflicts**, kemudian pilih kode yang akan diimplementasikan pada master branch.
3. Klik **Mark as resolved**, kemudian klik **Commit merge**.

## Istilah Branch
1. Branch 			= Jalur 'development' bebas dari sebuah commit
2. Checkout			= Berpindah ke branch / commit yang lain
3. Pull Request 	= Meminta pemilik repo untuk 'mengambil' perubahan yang telah dilakukan
4. Merge 			= Menggabungkan 2 branch
5. Merge Conflict 	= Baris yang sama diubah oleh 2 branch yang berbeda
