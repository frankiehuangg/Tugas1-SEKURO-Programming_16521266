## Definisi Fork
Forking pada Github didefinisikan sebagai sebuah kegiatan untuk mengambil repo orang lain beserta dengan historynya, dengan mencantumkan sumber dari repo tersebut. Forking digunakan untuk melakukan modifikasi terhadap repo original dan berkontribusi pada repo orang lain. Ingat bahwa **Fork** tidak sama dengan **Clone**.

## Melakukan Fork pada Repo Lain dengan Mengedit File
Langkah-langkah melakukan fork pada repo orang lain dengan mengedit salah satu file pada repo tersebut:
1. Pada halaman repo, klik file yang akan diedit isinya.
2. Klik tombol pensil **Fork this project and edit the file**.  

Setelah perubahan di commit, repo akan otomatis berada pada profil kita.

## Melakukan Fork Secara Manual
Untuk melakukan fork secara manual, pada halaman repo, klik **Fork** dan repo akan otomatis berada pada akun kita. Selain itu, kita juga dapat mendownload file yang ada pada repo dengan mengklik **Clone or download**. Perhatikan bahwa file yang telah didownload tidak akan memiliki informasi asal file.

## Melakukan Pull Request pada Repo Hasil Fork
Langkah-langkah melakukan pull request pada repo yang telah difork:
1. Pada halaman repo, klik **New pull request**.
2. Pilih branch dan repo tujuan, kemudian pastikan berstatus **Able to merge**.
3. Klik **Create pull request**, kemudian masukkan judul dan pesan kepada pemilik repo.
4. Klik **Create pull request**.  

Setelah dilakukan pull request, pemilik branch utama akan menerima request yang diberikan. Setelah diterima, nama kontributor akan muncul pada halaman repo yang bersangkutan.
