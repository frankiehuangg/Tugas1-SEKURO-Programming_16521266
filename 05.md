## Menginstall Git
Langkah-langkah menginstall Git pada Windows:
1. Kunjungi website utama [Git](https://git-scm.com)
2. Pada gambar monitor, klik tombol **Download [versi git] for Windows**.
3. Tunggu hingga selesai, kemudian launch installer tersebut.
4. Lanjutkan proses install pada installer, kemudian tunggu hingga selesai.

Untuk mengecek apakah git sudah terinstall pada Windows, dapat dilakukan dengan cara berikut:
1. Klik start menu, kemudian klik cmd.
2. Pada window yang muncul, ketik 'git' kemudian enter.

Jika tulisan *'git' is not recognized as an internal or external command, operable program or batch file.*, maka Git telah terinstall pada mesin.

Command untuk melihat versi git yang terpasang

Beberapa command Git (local) yang sering digunakan antara lain:
- `git init`
- `git add <file(s)>`
- `git status`
- `git commit`
- `git config`
- `git branch`
- `git help`

Saat dibuat sebuah repo, git akan membuat 3 "area" pada repo tersebut:
- Working tree  
  Folder tempat source-code berada.
- Staging area
  Area pada saat perubahan telah di inisiasi.
- History
  Area pada saat perubahan telah dilakukan.

**Staging area** dan **History** akan tersimpan pada file `.git`.

Untuk memindahkan perubahan dari **Working tree** menuju **Staging area**, kita akan melakukan `git add` dan untuk memindahkan **Staging area** menuju **History**, kita akan melakukan `git commit`.

## `git init`
Untuk mengubah folder saat ini menjadi repo, kita bisa menggunakan command `git init`.

![Menginisialisasi git](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/08.png)

Setelah diubah menjadi repo, akan terbuat sebuah folder tersembunyi `.git` yang berisi semua perubahan yang tersimpan oleh git. 

## `git status` dan `git add`
Untuk melihat status git, kita dapat menggunakan command `git status`. File yang belum pernah di-add akan berwarna merah dan perlu di add agar terekam oleh git.

![Status git](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/09.png)

## `git commit`
Sebelum melakukan commit, kita perlu memberikan identitas diri terlebih dahulu agar git dapat mengetahui siapa yang melakukan commit tersebut. Untuk melakukan pendaftaran diri, kita akan menggunakan command `git config --global user.email <email>` dan `git config --global user.name <nama>` .

![user.name](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/14.png)
![user.email](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/15.png)

Setelah itu, kita dapat melakukan commit dengan menggunakan command `git commit -m "<pesan>"`.

![`git commit`](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/16.png)
![Status git setelah `git commit`](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/17.png)

## `git log`
Untuk melihat semua perubahan yang telah dilakukan, dapat digunakan command `git log`.

![`git log`](https://raw.githubusercontent.com/frankiehuangg/Tugas1-SEKURO-Programming_16521266/main/pict/18.png)

Untuk melihat log n terakhir, dapat digunakan command `git log -<n>`. Untuk melihat log suatu file spesifik, dapat digunakan command `git log -- <file name>`.

## `git checkout`
Untuk melihat perubahan-perubahan sebelumnya, dapat digunakan command `git checkout <5 digit pertama hash> <tipe perubahan>`.
